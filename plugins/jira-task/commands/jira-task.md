# Jira Task 자동 생성

사용자의 기능/개선 요청을 정해진 포맷으로 정리한 뒤, 확인을 거쳐 Jira(Cloud)에 `Task` 티켓을 생성한다.

## 입력

사용자가 전달한 설명: `$ARGUMENTS`

## 절차

다음 단계를 **순서대로** 따른다. 절대 확인 없이 티켓을 생성하지 않는다.

### 1단계 — 요청 내용 파악

- `$ARGUMENTS` 에 내용이 **있으면**: 그 설명을 요청 내용으로 사용한다.
- `$ARGUMENTS` 가 **비어 있으면**: 지금까지의 대화 맥락과 최근 코드 변경(필요하면 `git diff`, `git log --oneline -5` 확인)을 근거로 무엇을 만들/개선하려는지 추론한다. 맥락이 불충분해 추론이 어려우면, 임의로 만들지 말고 사용자에게 무엇에 대한 티켓인지 한 번 되묻는다.

### 2단계 — 포맷에 맞춰 초안 작성

아래 포맷으로 **제목**과 **본문**을 작성한다.

- 제목 접두사: 새 기능이면 `[기능]`, 기존 동작 개선이면 `[개선]` 으로 분류한다.
- 본문은 Jira 위키 마크업 문자열로 그대로 들어가므로, 아래 구조를 유지한다.

```
제목: [기능] 또는 [개선] 한 줄 요약

h2. 배경 / 목적
왜 필요한지 (1~3줄)

h2. 요청 내용
무엇을 만들/바꿀 것인지 구체적으로

h2. 완료 조건 (AC)
* 조건 1
* 조건 2

h2. 참고 자료
관련 링크, 파일 경로, 스크린샷 등 (없으면 "없음")
```

작성한 **제목과 본문 전체를 사용자에게 보여주고**, "이대로 생성할까요? (수정할 부분이 있으면 알려주세요)" 라고 묻는다.

### 3단계 — 사용자 확인

- 사용자가 수정을 요청하면 반영 후 다시 2단계처럼 보여주고 재확인한다.
- 사용자가 거부하면 티켓을 생성하지 않는다.
- 사용자가 승인하면 4단계로 진행한다.

### 4단계 — 티켓 생성

1. 확정된 **본문**을 임시 파일에 저장한다. 예: `Write` 로 `/tmp/jira-task-desc.md` 에 본문 작성.
2. 플러그인에 포함된 스크립트를 실행한다 (`${CLAUDE_PLUGIN_ROOT}` 는 플러그인 루트로 치환됨):

   ```bash
   "${CLAUDE_PLUGIN_ROOT}/scripts/create-jira-task.sh" \
     --summary "확정된 제목" \
     --description-file /tmp/jira-task-desc.md
   ```

3. 스크립트 출력 결과를 사용자에게 그대로 전달한다.
   - 성공: 생성된 티켓 키와 URL 안내.
   - 실패(환경변수 누락·인증·권한 등): 스크립트가 알려준 원인과 해결 방법을 안내한다. 환경변수 미설정이면 `JIRA_EMAIL`, `JIRA_API_TOKEN` 설정 방법을 알려준다.

## 주의

- 제목/본문을 임의로 과장하거나 없는 요구사항을 만들어내지 않는다. 근거가 있는 내용만 적는다.
- 사용자 승인 전에는 어떤 경우에도 스크립트를 실행하지 않는다.
