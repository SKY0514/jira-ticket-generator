# Jira 버그 티켓 자동 생성

사용자가 보고한 버그/에러를 버그 전용 포맷으로 정리한 뒤, 확인을 거쳐 Jira(Cloud)에 티켓을 생성한다.

## 입력

사용자가 전달한 설명: `$ARGUMENTS`

## 절차

다음 단계를 **순서대로** 따른다. 절대 확인 없이 티켓을 생성하지 않는다.

### 1단계 — 버그 내용 파악

- `$ARGUMENTS` 에 내용이 **있으면**: 그 설명을 버그 내용으로 사용한다.
- `$ARGUMENTS` 가 **비어 있으면**: 지금까지의 대화 맥락과 최근 코드 변경/에러 로그(필요하면 `git diff`, `git log --oneline -5` 확인)를 근거로 어떤 버그인지 추론한다. 맥락이 불충분하면 임의로 만들지 말고 사용자에게 무엇에 대한 버그인지 되묻는다.

### 2단계 — 버그 포맷에 맞춰 초안 작성

아래 포맷으로 **제목**과 **본문**을 작성한다.

- 제목 접두사는 `[버그]` 로 한다.
- 본문은 Jira 위키 마크업 문자열로 그대로 들어가므로, 아래 구조를 유지한다.
- 재현 절차/기대·실제 동작을 모르면 추측해 채우지 말고, 해당 항목에 "확인 필요"로 남기거나 사용자에게 질문한다.

```
제목: [버그] 한 줄 요약

h2. 현상
무엇이 잘못 동작하는지 (1~3줄)

h2. 재현 절차
# 첫 번째 단계
# 두 번째 단계

h2. 기대 동작
정상적이라면 어떻게 동작해야 하는지

h2. 실제 동작
지금 실제로 어떻게 동작하는지 (에러 메시지/로그 있으면 포함)

h2. 참고 자료
관련 링크, 파일 경로, 스크린샷 등 (없으면 "없음")
```

작성한 **제목과 본문 전체를 사용자에게 보여주고**, "이대로 생성할까요? (수정할 부분이 있으면 알려주세요)" 라고 묻는다.

### 3단계 — 사용자 확인

- 사용자가 수정을 요청하면 반영 후 다시 2단계처럼 보여주고 재확인한다.
- 사용자가 거부하면 티켓을 생성하지 않는다.
- 사용자가 승인하면 4단계로 진행한다.

### 4단계 — 티켓 생성

1. 확정된 **본문**을 임시 파일에 저장한다. 예: `Write` 로 `/tmp/jira-bug-desc.md` 에 본문 작성.
2. 플러그인에 포함된 스크립트를 실행한다 (`${CLAUDE_PLUGIN_ROOT}` 는 플러그인 루트로 치환됨):

   ```bash
   "${CLAUDE_PLUGIN_ROOT}/scripts/create-jira-ticket.sh" \
     --summary "확정된 제목" \
     --description-file /tmp/jira-bug-desc.md
   ```

3. 스크립트 출력 결과를 사용자에게 그대로 전달한다.
   - 성공: 생성된 티켓 키와 URL 안내.
   - 실패(환경변수 누락·인증·권한 등): 스크립트가 알려준 원인과 해결 방법을 안내한다. 환경변수 미설정이면 `JIRA_EMAIL`, `JIRA_API_TOKEN` 설정 방법을 알려준다.

## 주의

- 재현 절차·실제 동작을 임의로 지어내지 않는다. 근거가 있는 내용만 적고, 모르면 "확인 필요"로 남긴다.
- 사용자 승인 전에는 어떤 경우에도 스크립트를 실행하지 않는다.
